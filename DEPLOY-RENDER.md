# Deploying to Render

Step-by-step to get the Metfraa Expense Portal live at **https://expenses.metfraa.com**.

The project ships with a `render.yaml` blueprint that defines the web service, a persistent disk, and all the environment variables. You only have to paste in the secrets.

---

## 0. Prerequisites

- The project pushed to a **GitHub repository** (Render deploys from GitHub).
- A **Render account** (render.com — sign up free).
- Your **Google OAuth** credentials (Client ID + Secret).
- Your **Azure / Microsoft Graph** credentials (Tenant ID, Client ID, Client Secret) — already created as the "Metfraa-Reimbursements" app.
- Access to DNS for **metfraa.com** (to point the subdomain).

---

## 1. Push the code to GitHub

```bash
cd bsg-portal
git init
git add .
git commit -m "Metfraa Expense Portal"
git branch -M main
git remote add origin https://github.com/<your-org>/metfraa-expense-portal.git
git push -u origin main
```

> `.gitignore` already excludes `.env`, `node_modules/`, and the local `data/`+`uploads/` folders, so no secrets or local data get pushed.

---

## 2. Create the service on Render (Blueprint)

1. In Render: **New → Blueprint**.
2. Connect your GitHub account and pick the repo.
3. Render reads `render.yaml` and shows the service **metfraa-expense-portal** with a 5 GB disk. Click **Apply**.
4. The first build runs `npm install` (this compiles `better-sqlite3` and `pdf-lib` — takes a couple of minutes). It will start, but **sign-in won't work yet** until you add the secret env vars (next step).

---

## 3. Add the secret environment variables

In the Render dashboard → your service → **Environment**, set these (they were left blank in the blueprint on purpose):

| Key | Value |
|---|---|
| `GOOGLE_CLIENT_ID` | your Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | your Google OAuth client secret |
| `MS_TENANT_ID` | `f4f5c484-2983-4c4c-aba3-7c9a0e1fddbc` |
| `MS_CLIENT_ID` | `3b75f62e-cb1b-4858-83b3-0842af465794` |
| `MS_CLIENT_SECRET` | the **Value** from your Azure client secret |

`SESSION_SECRET` is auto-generated by Render. The non-secret config (admin emails, OneDrive target = `info@metfraa.com`, etc.) is already baked into `render.yaml` — adjust there if needed.

Click **Save Changes** — Render redeploys automatically.

---

## 4. Point the domain

1. Render dashboard → your service → **Settings → Custom Domains → Add** `expenses.metfraa.com`.
2. Render shows you a CNAME target (like `metfraa-expense-portal.onrender.com`).
3. In your DNS (wherever metfraa.com is managed), add a **CNAME record**:
   - Host/Name: `expenses`
   - Value: the `…onrender.com` target Render gave you
4. Wait for DNS to propagate (minutes to an hour). Render auto-issues the HTTPS certificate.

> `APP_URL` in `render.yaml` is already set to `https://expenses.metfraa.com`. If you use a different subdomain, update `APP_URL` to match.

---

## 5. Update the Google OAuth redirect URI (critical)

In **Google Cloud Console → Credentials → your OAuth client → Authorized redirect URIs**, the entry MUST be **exactly**:

```
https://expenses.metfraa.com/auth/google/callback
```

It has to match `APP_URL` character-for-character. This is the single most common cause of "sign-in fails after deploy."

Also add `https://expenses.metfraa.com` under **Authorized JavaScript origins**.

---

## 6. Seed the employees (one time)

The database starts empty. Open the Render **Shell** (dashboard → your service → **Shell** tab) and run:

```bash
npm run seed
```

This loads all 31 Metfraa employees onto the persistent disk. Re-running is safe (it skips existing ones). You can also seed from the CSV: `npm run seed -- metfraa-employees.csv`.

---

## 7. Verify (the dry run)

1. Visit `https://expenses.metfraa.com` → sign in with a seeded employee's Google account.
2. Submit a test claim with a bill.
3. Check `info@metfraa.com`'s OneDrive → `Reimbursements and Conveyance/<Employee>/` should appear with `Uploads/` populated and `_Log.xlsx` created.
4. Sign in as `info@metfraa.com` → Admin Panel → **Pending** → Approve.
5. Confirm the merged PDF lands in that employee's `Reports/` folder and the Excel row flips to **Approved**.

If all five happen, you're live.

---

## Ongoing notes

- **Deploys**: every `git push` to `main` auto-redeploys. The persistent disk keeps `data/` and `uploads/` intact across deploys.
- **Backups**: periodically download `data/portal.db` from the Render Shell (`/var/data/data/portal.db`), or take a Render disk snapshot. This is your entire submission history.
- **Logs**: Render dashboard → **Logs**. OneDrive sync errors (if any) show up here; the app retries automatically every 15 minutes.
- **Scaling**: the Starter plan is plenty for 31 employees. No action needed.
- **Cost**: Starter web service (~$7/mo) + 5 GB disk (~$1.25/mo). Around $8–9/month total.
